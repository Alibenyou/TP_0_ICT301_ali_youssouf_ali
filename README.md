## LES PRINCIPES SOLID
Voici ce que j'ai retenu de la mise en œuvre de ces principes :

1. **SRP (Single Responsibility Principle) :** 
            Une classe doit avoir une seule responsabilité. Cela évite qu'une modification dans la logique métier ne casse la logique d'affichage ou de sauvegarde.
2. **OCP (Open-Closed Principle) :** 
            Le code doit être ouvert à l'extension (ajout de nouvelles formes/fonctions) mais fermé à la modification (ne pas toucher au code existant qui fonctionne).
3. **LSP (Liskov Substitution Principle) :** 
                Une classe enfant doit pouvoir remplacer sa classe parente sans créer d'erreurs inattendues.
4. **ISP (Interface Segregation Principle) :** 
                Il vaut mieux créer plusieurs petites interfaces spécifiques plutôt qu'une seule grande interface qui force les classes à implémenter des méthodes dont elles n'ont pas besoin.
5. **DIP (Dependency Inversion Principle) :**    
                Il faut dépendre des abstractions (interfaces) et non des implémentations concrètes pour rendre le système flexible.
